KVOController

发表于 2018-05-11 | 分类于 斑马日报 2018/05/11 第11期 | | 阅读次数:

适合: 初级开发者阅读
来自: Github
关键字: KVO, Key-Value Observing, 第三方框架
原文：KVOController

KVO 作为 iOS 中一种强大并且有效的机制，为iOS开发者们提供了很多的便利；我们可以使用KVO来检测对象属性的变化、快速做出响应，这能够为我们在开发强交互、响应式应用以及实现视图和模型的双向绑定时提供大量的帮助。但是在大多数情况下，除非遇到不用KVO无法解决的问题，我都会尽量避免它的使用，这并不是因为KVO有性能问题或者使用场景不多，重要的原因是KVO的使用是在是太坑了。使用 KVO 时，既需要进行注册成为某个对象属性的观察者，还要在合适的时间点将自己移除，再加上需要覆写一个又臭又长的方法，并在方法里判断这次是不是自己要观测的属性发生了变化，每次想用 KVO 解决一些问题的时候，我的第一反应就是头疼。

那如何优雅的使用KVO呢？Facebook为我们开源了KVOController框架, 使用KVOController进行键值观测可以说完美地解决了在使用原生KVO时遇到的各种问题。

  • 不需要手动移除观察者；
  • 实现 KVO 与事件发生处的代码上下文相同，不需要跨方法传参数；
  • 使用 block 来替代方法能够减少使用的复杂度，提升使用 KVO 的体验；
  • 每一个 keyPath 会对应一个属性，不需要在 block 中使用 if 判断 keyPath；

所以，如果你考虑使用KVO，那么不妨试试KVOController吧，会让你觉得豁然开朗。

适合: 初级开发者阅读
来自: developer.apple.com
关键字: KVO, Key-Value Observing
原文：Key-Value Observing Programming Guide

Key-Value Observing, KVO, 键值观察，这是一种机制，当指定对象属性改变的时候允许另一个对象接受通知。由NSObject的一个非正式协议NSKeyValueObserving(其实就是NSObject的一个Category)实现。这相当于NSObject及其子类自带了设计模式中的观察者模式。 这就意味着我们不需要自己去实现这样的方案。每次属性改变了就发送消息通知，这是KVO机制提供的最大的优点，因为这个方案已经被明确定义，获得框架级支持，可以方便地采用。

想要了解KVO，首先要推荐的当然是苹果的官方文档：Key-Value Observing Programming Guide

不知道为什么Apple开始废弃一系列Programing Guide的更新。在新版的developer.apple.com，我们已经找不到这篇文章了，我们可以在archive中找到，但这并不是说这篇文章过时了，它依旧很有指导意义，帮助你了解KVO编程需要注意的事。所以，如果你的英文还可以的话，那么这是你必须要阅读的关于KVO的第一篇文章。

适合: 初级开发者阅读
来自: developer.apple.com
关键字: Key-Value Coding, KVC
原文：Key-Value Coding Programming Guide

Key-Value Coding, KVC, 键值编码，这是我们在日常编码中经常会用到的一种技术。这得益于OC的语言特性，可以让我们不必进行任何操作就可以进行属性的动态读写。KVC的操作方法由NSKeyValueCoding非正式协议提供，而NSObject以Category的形式实现了这个非正式协议，因此可以说OC中几乎所有的对象都支持KVC操作，正如你常用到的， KVC常用的操作方法如下：

  • 动态设置：setValue:forKey:, setValue:forKeyPath:
  • 动态读取：valueForKey:, valueForKeyPath:

那么你知道它的实现原理吗？它还有哪些别的功能吗？又或者我们在使用KVC编码时需要注意什么？它的性能又如何？Apple给出了答案，那就是推荐给大家的Key-Value Coding Programming Guide 。

不知道为什么Apple开始废弃一系列Programing Guide的更新。在新版的developer.apple.com，我们已经找不到这篇文章了，我们可以在archive中找到，但这并不是说这篇文章过时了，他依旧很有指导意义，帮助你了解KVC背后的故事。所以，如果你的英文还可以的话，那么这是你必须要阅读的关于KVC的第一篇文章。
